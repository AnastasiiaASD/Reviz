# Instruction: Write Tests

## Objective
Write Playwright tests based on Phase 2 scenarios.
Create isolated folder `tests/{{task-key}}/` in the cloned repo.
Do NOT push or create PR — the orchestrator handles that after you exit.

Read @/instructions/generic/project-description.md before start.
Knowledge base: /app/knowledge-base/ — read relevant feature file.

## ⛔ Token budget
- Read `/app/tmp/{{task-key}}_details.txt` — НЕ викликати jira-ai task-with-details
- Max 2 playwright-cli snapshots якщо потрібен браузер
- НЕ клонувати репо — вже є в /app/repo
- НЕ робити git push, НЕ створювати PR — оркестратор зробить після виходу

## Workflow

### 1. Read context

```bash
cat /app/tmp/{{task-key}}_details.txt
```

Прочитай Jira коментарі — знайди сценарії з Phase 2 (лейбла `scenarios-done`).
Перевір `tests/regression/` на наявність пов'язаних тестів — можна розширити замість створення нових.

### 2. Підготувати гілку і папку

```bash
cd /app/repo
git checkout -b tests/{{task-key}}
mkdir -p tests/{{task-key}}
```

### 3. Написати spec файл

Файл: `tests/{{task-key}}/{{task-key}}.spec.js`

Правила POM:
- Використовувати існуючі Page Objects з `tests/pages/`
- Якщо потрібного Page Object немає — створити в `tests/pages/`
- Naming: `should [action] when [condition]`
- Без `waitForTimeout`, без shared state між тестами
- Credentials через `process.env`, не хардкодити
- Кожен тест незалежний (beforeEach для setup)

### 4. Написати README

Файл: `tests/{{task-key}}/README.md`

Зміст README:
- Заголовок: `# Tests: {{task-key}}`
- Секція `## Що тестується` — 1-2 речення з описом фічі
- Секція `## Jira` — `{{task-key}}: [назва тікету]`
- Секція `## Запуск локально` — команда `npx playwright test tests/{{task-key}}/`
- Секція `## Змінні середовища` — `BASE_URL` (URL середовища), `TEST_USER_EMAIL` / `TEST_USER_PASSWORD` (тестові credentials)

### 5. Закомітити

```bash
cd /app/repo
git add tests/{{task-key}}/
git commit -m "test({{task-key}}): автотести для {{task-key}}

Auto-generated by Reviz AI Agent"
```

Оркестратор сам зробить git push і відкриє PR.
